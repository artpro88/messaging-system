# Step-by-Step Fly.io Deployment\n\n## Fixed Issue\n\nRemoved root `fly.toml` that was causing conflicts. Now using `backend/fly.toml` as the source of truth.\n\n## Step 1: Create PostgreSQL Database\n\n**Go to**: https://fly.io/app/dashboard\n\n1. Click **New App** button (top right)\n2. Select **Postgres** from the list\n3. Choose configuration:\n   - **App Name**: `messaging-db`\n   - **Region**: London (lhr)\n   - **Configuration**: Development (free - 256MB RAM, 1GB disk)\n4. Click **Create Database**\n5. **WAIT** - It will show you the connection credentials\n6. **COPY THIS** - You'll see something like:\n   ```\n   Database hostname: messaging-db.flycast\n   Database name: postgres\n   Database user: postgres\n   Database password: [long string]\n   ```\n7. **Create the connection string**:\n   ```\n   postgresql://postgres:PASSWORD@messaging-db.flycast:5432/postgres\n   ```\n   (Replace PASSWORD with the actual password shown)\n\n## Step 2: Set Database Secret in Backend App\n\n1. Go back to dashboard\n2. Click on `messaging-system-backend` app\n3. Go to **Secrets** tab (left sidebar)\n4. Click **Add Secret**\n5. **Name**: `DATABASE_URL`\n6. **Value**: (paste the connection string from Step 1)\n7. Click **Add**\n\n## Step 3: Deploy Backend from GitHub\n\n**Option A: Using Dashboard (EASIEST)**\n\n1. Click on `messaging-system-backend` app\n2. Go to **Deployments** tab\n3. You should see GitHub integration\n4. If not connected:\n   - Click **GitHub** under integrations\n   - Authorize Fly.io to access your GitHub\n   - Select repo: `artpro88/messaging-system`\n5. Select branch: `main`\n6. Click **Deploy**\n7. Wait for build and deployment (5-10 minutes)\n\n**Option B: Using Command Line**\n\nFrom `backend/` directory:\n```bash\ncd backend\nflyctl deploy\n```\n\n## Step 4: Verify Deployment\n\n1. In Fly.io dashboard, go to `messaging-system-backend`\n2. Click **Overview** tab\n3. Look for **Hostname**: Should be `messaging-system-backend.fly.dev`\n4. Open in browser: https://messaging-system-backend.fly.dev/health\n5. Should show: `{\"status\":\"ok\"}`\n\n**If it shows error**:\n- Check **Logs** tab for error messages\n- Common issue: DATABASE_URL not set\n- Solution: Set the secret and redeploy\n\n## Step 5: Update Vercel Frontend\n\n**Go to**: https://vercel.com/dashboard\n\n1. Select your project `messaging-system-frontend`\n2. Click **Settings** (top menu)\n3. Go to **Environment Variables** (left sidebar)\n4. Find or create:\n   - `NEXT_PUBLIC_API_URL` = `https://messaging-system-backend.fly.dev`\n   - `NEXT_PUBLIC_WS_URL` = `https://messaging-system-backend.fly.dev`\n5. Click **Save**\n6. Go to **Deployments** tab\n7. Click the three dots on latest deployment\n8. Select **Redeploy**\n9. Wait for deployment to complete\n\n## Step 6: Test Everything\n\n1. Open: https://messaging-system-frontend.vercel.app/demo\n2. Fill the form:\n   - Name: \"Test\"\n   - Email: \"test@example.com\"\n   - Message: \"Hello!\"\n3. Click Send\n4. Open new tab: https://messaging-system-frontend.vercel.app/inbox\n5. **Check**: Conversation appears immediately ✅\n6. Click the conversation\n7. **Check**: Channel shows \"💭 Live Chat\" (not Email) ✅\n8. Type reply: \"Hi there!\"\n9. Click Send\n10. Go back to demo tab\n11. **Check**: Reply appears in chat widget ✅\n\n## Troubleshooting\n\n### \"Connection refused\" when visiting health endpoint\n- Backend not started: Check Logs in Fly.io dashboard\n- Wrong hostname: Verify hostname in Overview tab\n- Still building: Wait 5-10 minutes\n\n### \"Cannot connect to database\"\n- DATABASE_URL not set: Go to Secrets tab and add it\n- Wrong connection string: Verify it matches the one from Step 1\n- Solution: Set secret and redeploy\n\n### Real-time features still not working\n- Frontend not updated: Verify Vercel env variables are set\n- Vercel not redeployed: Redeploy from Deployments tab\n- Clear browser cache: Ctrl+Shift+Delete\n- Try incognito window\n\n## Quick Reference\n\n**Backend**: https://messaging-system-backend.fly.dev\n**Frontend**: https://messaging-system-frontend.vercel.app\n**Fly Dashboard**: https://fly.io/app/dashboard\n**Vercel Dashboard**: https://vercel.com/dashboard\n\n**Env Variables Needed**:\n- DATABASE_URL (set in Fly.io Secrets)\n- JWT_SECRET (already set)\n- FRONTEND_URL (already set)\n- NEXT_PUBLIC_WS_URL (set in Vercel)\n- NEXT_PUBLIC_API_URL (set in Vercel)\n"}
